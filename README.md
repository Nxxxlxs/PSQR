# Product Quality Assessment - Estudo de Caso em QA

## Visão Geral

Este repositório contém um estudo prático de QA focado em análise funcional, testes exploratórios, documentação de defeitos e gerenciamento de issues realizados sobre duas aplicações web.

O objetivo deste projeto foi simular um fluxo real de atuação de QA, identificando problemas nos produtos, avaliando seus impactos e documentando os achados utilizando práticas estruturadas de rastreamento de defeitos.

Mais do que apenas identificar bugs, este repositório demonstra uma abordagem voltada para qualidade envolvendo:

- Análise funcional
- Testes exploratórios
- Avaliação de riscos
- Triagem de defeitos
- Visão de produto
- Organização de backlog
- Documentação técnica
- Avaliação de experiência do usuário
- Classificação de severidade e prioridade

---

## Objetivo do Projeto

Este projeto foi originalmente desenvolvido como parte de uma avaliação técnica para uma vaga de QA e posteriormente refinado como estudo de caso para portfólio profissional.

O repositório demonstra como análises de qualidade podem ser estruturadas em um contexto próximo ao ambiente corporativo utilizando GitHub Issues como plataforma de gerenciamento de defeitos.

## Estrutura do Repositório

Este repositório utiliza GitHub Issues como principal artefato de rastreamento e análise de defeitos.

Cada issue pode conter:

- Caminho do produto alvo
- Tipo
- Classificação
- Prioridade
- Descrição Geral
- Descrição Objetiva
- Critérios de aceite
- Evidências
- Resultado desejável quando aplicável

---

## Metodologia Utilizada

Durante a análise foram considerados diversos aspectos relacionados à qualidade do produto, incluindo:

- Consistência funcional
- Consistência visual
- Impacto na experiência do usuário
- Fluxos de navegação
- Comportamentos de validação
- Responsividade
- Acessibilidade

O objetivo não foi apenas encontrar falhas, mas também avaliar sua relevância e impacto na qualidade geral do produto.

---

## Tipo

O tipo define o que será implementado na tarefa em questão.

| Tipo                | Descrição                                                      |
| ------------------- | ---------------------------------------------------------------- |
| Correção          | Requer reparo para retomada de funcionalidades.                  |
| Nova funcionalidade | Requer o desenvolvimento de novos recursos ou funções.         |
| Melhoria            | Requer otimização e aprimoramento de experiência do usuário. |

---



## Classificação

A classificação define a categoria na qual o problema descrito se encaixa .

| Classificação | Descrição                                                 |
| --------------- | ----------------------------------------------------------- |
| Utilidade       | Impedem a execução de tarefas e funções da aplicação. |
| Usabilidade     | Dificultam ou atrasam o uso da aplicação.                 |
| Desejabilidade  | Oportunidade de melhoria da aplicação.                    |

---



## Prioridade

A prioridade define a urgência da correção considerando impacto no produto e no negócio.

| Prioridade | Descrição                                                               |
| ---------- | ------------------------------------------------------------------------- |
| Alta       | Problema de alto impacto que impede o uso da aplicação.                 |
| Média     | Problema que compromete significativamente o uso da aplicação.          |
| Baixa      | Problema de menor urgência que pode ser verificado em momento oportuno. |
